# PR Plan: Phase101-110

## 設計ステータス（2026-03-05）

- PR101-PR110: 設計確定（実装は次スプリントで段階投入）

`docs/ROADMAP.md` と `docs/06_phase_backcast_1_100.md` の残タスクを起点に、  
Phase101-110 では GA hardening の未完了項目を「実運用で壊れない」水準まで引き上げます。

## 方針

- 追加機能よりも、既存機能の信頼性と監査性を優先して仕上げる
- plugin / publish / release / LTS を個別最適ではなく、一連の運用フローとして接続する
- `verify-v1` を推奨チェックから必須ゲートへ移行し、互換性逸脱を早期に止める

## Phase110完了時のDefinition of Done

- plugin sandboxがOS隔離込みで運用可能になり、契約テストで継続検証できる
- SDK多言語テンプレートと`plugin init`で第三者拡張の立ち上げが短縮される
- webhook/notification/provider連携が重複排除付きで再送可能になる
- `verify-v1` が実運用向けの厳格判定を提供し、CIゲートへ組み込める
- release artifactが実署名 + 標準SBOMで監査可能になり、LTS backportが半自動化される
